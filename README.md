# Retail Inventory Management System - Global Auto Parts

A high-performance, professional command center for managing retail inventory, sales transactions, and financial reporting. Built for **Global Auto Parts**, this system leverages a modern Next.js frontend with a robust Google Sheets backend powered by Google Apps Script.

## 🚀 Key Features

### 📊 Intelligent Dashboard
- **Real-time Analytics**: Monitor total stock value, cash balance, and today's sales at a glance.
- **Smart Alerts**: Visual indicators for low-stock items with progress bars and status badges.
- **Activity Feed**: Live feed of recent transactions for immediate oversight.

### 📦 Inventory Management
- **Category Autocomplete**: Intelligent suggestions based on existing stock to ensure consistent data entry.
- **Dynamic Stock Tracking**: Automated status updates (Active, Low Stock, Out of Stock) based on custom thresholds.
- **SKU Management**: Simplified product tracking with dedicated SKU and category fields.

### 🛒 Advanced Transactions
- **Batch Processing**: Record multiple products in a single order (Shopping Cart style).
- **Searchable Selector**: Instantly find products by Name or SKU with real-time stock availability check.
- **Dual Type Support**: Handle both "Sales" (Stock Out) and "Restocking" (Stock In) seamlessly.

### 📄 Professional Invoicing & Reporting
- **Automated PDF Invoices**: Generate highly formatted, premium invoices using `jsPDF`.
- **Customizable Invoices**: Optional client details, tax calculations, and persistent company branding.
- **Financial Ledger**: Complete history of debits, credits, and balance snapshots.

## 🛠️ Technology Stack

- **Frontend**: Next.js 15 (App Router), React, Tailwind CSS
- **Icons**: Lucide React
- **Data Fetching**: SWR (Stale-While-Revalidate) for real-time synchronization
- **PDF Engine**: jsPDF & jsPDF-AutoTable
- **Backend/Database**: Google Apps Script & Google Sheets API

## ⚙️ Setup & Installation

### 1. Backend (Google Sheets & Apps Script)
1. Create a new Google Sheet.
2. Go to **Extensions > Apps Script**.
3. Copy the contents of `Code.gs` from this repository into the Apps Script editor.
4. Run the `setupDatabase` function once to initialize the sheets and headers.
5. Click **Deploy > New Deployment**.
   - Select **Web App**.
   - Set "Execute as" to **Me**.
   - Set "Who has access" to **Anyone**.
6. Copy the generated **Web App URL**.

### 2. Frontend (Next.js)
1. Navigate to the `frontend` directory:
   ```bash
   cd frontend
   ```
2. Install dependencies:
   ```bash
   npm install
   ```
3. Create a `.env.local` file in the `frontend` folder and add your API URL:
   ```env
   NEXT_PUBLIC_API_URL=YOUR_GOOGLE_APPS_SCRIPT_URL_HERE
   ```
4. Run the development server:
   ```bash
   npm run dev
   ```

## 🌐 Deployment (Netlify)

This project is optimized for deployment on **Netlify**.
1. Push the code to GitHub.
2. Connect the repository to Netlify.
3. Add the `NEXT_PUBLIC_API_URL` environment variable in the Netlify Dashboard.
4. The system will automatically build and deploy using the included `netlify.toml` configuration.
